# 🏦 Banking Analytics Project using SQL

## 📌 Project Objective  
To design a relational banking database and perform SQL-based analysis to extract KPIs, business insights, and basic risk indicators from structured banking data.

---

## 📊 Project Type  
Descriptive and diagnostic analytics using SQL.

---

## 📂 Data Source  
Structured CSV banking datasets:
- customers.csv  
- accounts.csv  
- loans.csv  
- transactions.csv  
- branches.csv  

---

## 🛠 Tools Used  
- SQL  
- Relational Database Design  
- Database Modeling  
- CSV Data Processing  
- Data Cleaning  
- Business Intelligence Queries  

---

## 🗄 Database Design  
- Normalized relational schema  
- Primary and foreign key relationships  
- Structured table design  
- Entity relationships between customers, accounts, loans, transactions, and branches  

---

## 📥 Data Processing  
- Data import using SQL scripts  
- Data cleaning and standardization  
- Consistency validation  
- Relational integrity checks  

---

## 📈 Analytics Performed  
- KPI generation (accounts, customers, loans, transactions)  
- Customer behavior analysis  
- Branch performance analysis  
- Transaction analysis  
- Loan distribution analysis  
- Rule-based risk pattern identification  

---

## 🔍 Business Problems Addressed  
- How customers are distributed across branches  
- How transactions vary across branches  
- Which customers show high transaction activity  
- How loans are distributed across the customer base  
- Which accounts maintain low balances  
- Which customers hold multiple loans  
- Which accounts/customers show unusual patterns  

---

## 📌 Key Insights  
- Customer and transaction distributions vary by branch  
- Transaction volumes are uneven across customers  
- Loan exposure differs significantly across the customer base  
- Some accounts consistently maintain low balances  
- Certain customers hold multiple active loans  
- Risk indicators are clustered in specific customer segments  

---

## 🩺 Recommendations   
- Monitor customers flagged by rule-based risk queries  
- Track accounts with low balance patterns  
- Review branches with high transaction concentration  
- Use KPI outputs for performance monitoring  
- Use risk queries for identifying unusual activity patterns  
- Use SQL reports for operational decision support
